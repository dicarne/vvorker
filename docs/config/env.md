# CLI 环境

VVorker CLI 支持多环境管理，每个环境可以配置不同的平台 URL 和 API 密钥，方便在不同环境（如开发、测试、生产）之间切换。

## 创建环境

使用 `vvcli create` 命令创建新环境：

```bash
vvcli create
```

命令会交互式地要求输入以下信息：

### 环境名称（envName）

- 用于标识不同的部署环境
- 示例：`dev`（开发环境）、`test`（测试环境）、`prod`（生产环境）
- 必须唯一，不能重复

### URL

- VVorker 平台的根地址
- 示例：`https://vvorker.cloud.zhishudali.ink`
- 应该是完整的 URL，包含协议（http/https）

### API 密钥

- 用于身份验证的访问令牌
- 从 VVorker 平台的用户页面获取
- 用于 API 请求的认证

::: tip 创建第一个环境
当您创建第一个环境时，它会自动被设置为当前环境。
:::

## 切换环境

使用 `vvcli env` 命令切换当前环境：

```bash
vvcli env
```

该命令会列出所有已创建的环境，您可以选择要切换到的环境。

## 使用场景

- **多环境部署**：为不同的环境（开发、测试、生产）配置不同的 URL 和 API 密钥
- **多平台管理**：连接到不同的 VVorker 平台实例
- **团队协作**：每个开发者可以使用自己的环境配置

::: warning 注意事项
- 如果没有创建任何环境，部署命令会报错，提示您先创建环境
- API 密钥 是敏感信息，请妥善保管
- 建议为不同环境使用不同的 API 密钥
:::
